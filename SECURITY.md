# Security Policy

## Reporting a Vulnerability
If you discover a security vulnerability in this project, please report it via GitHub Issues or contact the maintainer directly. 

### What to report
- Vulnerabilities that allow for Cross-Site Scripting (XSS).
- Vulnerabilities that bypass the architectural constraints enforced by the linter.
- Environment-specific security issues.

### Disclosure
We aim to acknowledge and address critical security reports within 7 days. Please do not publicize the vulnerability until a fix has been released.
